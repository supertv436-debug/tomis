# Развертывание на GitHub Pages

## 🚀 Быстрый старт

1. **Создайте репозиторий на GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_НИК/ВАШ_РЕПОЗИТОРИЙ.git
   git push -u origin main
   ```

2. **Активируйте GitHub Pages**
   - Перейдите в настройки репозитория
   - Sections → Pages
   - Source: Deploy from a branch
   - Branch: main
   - Folder: /root
   - Save

3. **Готово!** 🎉
   Ваш сайт будет доступен по адресу: `https://ВАШ_НИК.github.io/ВАШ_РЕПОЗИТОРИЙ`

## 📋 Что нужно сделать

### 1. Замените плейсхолдеры
В файле `_config.yml` замените:
```yaml
url: "https://[username].github.io"
```
на:
```yaml
url: "https://ВАШ_НИК.github.io"
```

### 2. Настройте автоматическое развертывание (опционально)
Если вы создали файл `.github/workflows/pages.yml`, то:
- Перейдите в Settings → Actions → General
- Включите "Allow GitHub Actions to create and approve pull requests"
- Включите "Allow GitHub Actions to run workflows from forked repositories"

## 🔧 Функционал

### ✅ Работает в браузере
- Чтение PDF и DOCX файлов
- Прямые запросы к Groq API
- Генерация документов
- Сохранение ключа в localStorage

### 🛡️ Безопасность
- API ключ хранится только в браузере
- Никаких серверных компонентов
- Все данные обрабатываются локально

## 📱 Особенности

- **Адаптивный дизайн** - работает на всех устройствах
- **Drag & Drop** - удобная загрузка файлов
- **Real-time чат** - мгновенные ответы ИИ
- **Offline ready** - работает без интернета (кроме API запросов)

## 🌐 CDN ресурсы

Сайт использует следующие CDN:
- Tailwind CSS
- Font Awesome
- PDF.js
- Mammoth.js

## 🔍 Проверка

После развертывания проверьте:
1. Главная страница загружается
2. Можно установить API ключ
3. Файлы загружаются корректно
4. Чат с ИИ работает
5. Генерация документов функционирует

## 🐛 Troubleshooting

### Если сайт не работает:
1. Проверьте что `_config.yml` в корне репозитория
2. Убедитесь что `index.html` тоже в корне
3. Проверьте настройки Pages в GitHub
4. Посмотрите логи сборки в Actions

### Если API не отвечает:
1. Проверьте валидность API ключа
2. Убедитесь что ключ начинается с `gsk_`
3. Проверьте подключение к интернету

## 📞 Поддержка

Если возникли проблемы:
1. Проверьте консоль браузера (F12)
2. Убедитесь что все CDN ресурсы загрузились
3. Проверьте настройки CORS в браузере

---

**Важно:** GitHub Pages - это статический хостинг. Все функции работают полностью в браузере клиента!
